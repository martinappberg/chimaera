# Native-shell rules

Depth: [chimaera-app/CLAUDE.md](../../crates/chimaera-app/CLAUDE.md).

- **This is a standalone cargo workspace.** Tauri stays out of the daemon workspace so
  musl/HPC builds stay lean; the daemon crates compile twice (intentional). Never fold
  the app into the root workspace. Root `cargo` doesn't build it — use `just app-*`.
- **The command surface is a quad-lockstep** the compiler does not enforce: `shell.rs`
  `generate_handler!` ↔ `build.rs` ↔ `capabilities/daemon-ui.json` ↔ committed
  `permissions/autogenerated/*.toml` (plus `native.ts` wrappers). Change a command in
  all of them, then `npx tauri build` to confirm they agree.
- **Version stamping matches the literal `0.0.1` sentinel** via `sed` across
  `tauri.conf.json` + both `Cargo.toml`s. A pre-bumped value makes the sed silently
  no-op and ship the wrong version.
- **`--askpass` must stay lightweight** — it's the `SSH_ASKPASS` relay and must never
  spawn a daemon or window. The three-role argv dispatch order in `main.rs` is
  load-bearing.
- Signing is release-only (`TAURI_SIGNING_PRIVATE_KEY*`); the `app.yml` PR build needs
  no key.
