An agent workbench, not an IDE.

chimaera runs your coding agents as persistent, daemon-owned sessions on whatever host owns the work — your laptop, a dev server, or an HPC login node — with a workspace-first UI around them. Close your laptop mid-run and the sessions keep going.

why it exists

One program instead of two half-fits

Running agents on a remote machine usually means two tools: code-server in a browser tab to look at files, and tmux over SSH to keep the agents alive. Browser terminals die on reload; tmux shows no file output. And the result of an agent run is usually files — reports, plots, tables — that neither tool puts next to the session that produced them. chimaera does both jobs, in one window, on the host that owns the work.

features

What's in it

Sessions survive disconnect

The daemon holds every PTY with full server-side terminal state. Reload, reconnect, or reattach from another machine and you get the same screen back — no lost scrollback, no broken reconnect.

One launcher for your agents

Claude Code, Codex, Gemini CLI and Antigravity — detected if installed, resumable per workspace, installable from official sources. Binaries live under ~/.chimaera; credentials stay yours.

The real agent TUIs

Agents run as the actual claude TUI in a daemon-owned PTY, so they look, behave, and bill exactly like they do in your terminal. Hooks surface which one needs you.

Previews that replace code-server

Images, Markdown, CSV/TSV (gzip included), PDF, notebooks, and sandboxed HTML reports — the MultiQC/FastQC class of output. The server extracts, the client renders; whole files never load.

A real workbench layout

Split panes, tabs, drag-and-drop, focus mode, and a session strip that says where you are. Terminals and file previews are the same kind of surface — arrange them however the work wants.

Linked terminals

Hand an agent a leash to a live shell — the one with your modules and conda environment already loaded — instead of paying setup cost per command. Scoped, user-granted, and audited in the shell's own scrollback.

how it works

The daemon owns the sessions. Windows are views.

local or remote

Connect over your own SSH

chimaera connect <host> shells out to your system ssh, so ProxyJump, ControlMaster, and 2FA from ~/.ssh/config all work as-is. The daemon installs into ~/.chimaera on the host — no root, no containers.

  • Install-if-missing. Detects the target with uname and drops in the right static binary.
  • Start-or-attach. Reuses a running daemon, tunnels the port, opens the UI.
  • Self-updating. An outdated remote daemon is replaced on connect — an idle shell holding an environment is never killed without asking.
zsh — your laptop 
# run it locally
$ chimaera serve
  → workbench on http://127.0.0.1:9700

# or drive a remote host — HPC login node, dev box, anything
$ chimaera connect gpu-node
  → installing daemon in ~/.chimaera … ok
  → tunnelling :9700 … ok
  → opening workbench
built for shared machines

One static binary. Old glibc. No root.

The daemon cross-compiles to fully static musl binaries (x86_64 and aarch64) that run on old HPC systems — verified on a cluster running glibc 2.17, no root. It's built to live on a shared login node: bounded allocations, hard ceilings on preview extraction, ~150 MB RSS steady-state.

  • NFS-safe state. Append-only, size-capped JSONL under ~/.chimaera — no SQLite near Lustre.
  • Good citizen. No busy loops, no unbounded buffers.
  • Native app included. A Tauri shell wraps the same UI with a home screen of workspaces and one-click connect. It auto-updates; quitting kills nothing.

faq

Good to know

Does it change how my agents run or bill?
No. Agents run as their real TUIs in a normal PTY, spawned through your login shell with your environment and credentials. claude behaves and bills exactly as it does in any terminal — chimaera just owns the process and gives you a window onto it.
Do my credentials go through chimaera?
No. Credentials stay yours — they live in your HOME and login-shell environment, independent of who installed the agent binary. chimaera can install and update agent CLIs from official sources, but never touches your auth.
Is it only for HPC?
No — it's a general tool for running agents wherever the work lives. HPC login nodes are the founding stress test (old glibc, no root, shared machines), which is why it also behaves well on a laptop or a plain dev server.
What platforms are supported?
macOS and Linux. The daemon ships as static musl binaries for x86_64 and aarch64 Linux plus an aarch64 macOS build; the native app is macOS (Apple Silicon). Windows is untested. See the docs.
